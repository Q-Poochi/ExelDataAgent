================================================================================
KIẾN TRÚC DỮ LIỆU DATAAGENT (DATA FLOW GUIDE)
================================================================================

Tai lieu nay giai thich cach van hanh "Bat dong bo - Realtime" cua he thong.
Day la mo hinh chuan cho cac ứng dung xu ly du lieu nang (AI, Big Data).

--------------------------------------------------------------------------------
1. CÁC THÀNH PHẦN CHÍNH
--------------------------------------------------------------------------------
- API (ASP.NET 8): Cua ngo tiep nhan moi yeu cau tu nguoi dung.
- MinIO: Kho luu tru file (Object Storage).
- Hangfire: Trình quan ly chay ngam (dam bao tac vu luon hoan thanh).
- n8n: Bo nao AI (noi thuc hien logic phan tich, goi Claude/ChatGPT).
- SignalR: Duong truyen tin nhan tuc thoi (Real-time push).
- Chart Service (Python): Chuyen gia ve bieu do va xuat PDF bao cao.

--------------------------------------------------------------------------------
2. QUY TRÌNH 5 BƯỚC (STEP-BY-STEP)
--------------------------------------------------------------------------------

BUO'C 1: TẢI FILE (UPLOAD)
- Ban gui file len API.
- API luu file vao MinIO va ghi thong tin metadata vao SQL Server.
- Ban nhan ve mot cái ID goi la "fileId".

BUO'C 2: KÍCH HOẠT PHÂN TÍCH (START ANALYSIS)
- Ban gui request kem "fileId" vao API.
- API tao mot Job trong database (trang thai Pending).
- API nem cong viec nay vao Hangfire (Run in background).
- Ban nhan ve ngay lap tuc "jobId" de theo doi (Khong phai cho doi).

BUO'C 3: GỌI N8N ( xỬ LÝ NGẦM )
- Hangfire tu dong chay class "TriggerN8NWorkflowJob".
- No tao mot link bi mat de n8n co the lay file tu MinIO.
- No gui lenh sang n8n (POST) kem "jobId", "fileUrl" va "callbackUrl".

BUO'C 4: AI PHÂN TÍCH ( TẠI N8N )
- n8n nhan lenh va bat dau chay Workflow AI cua ban.
- Trong qua trinh chay, n8n co the goi ve API de bao cao: "Toi dang lam X%...".

BUO'C 4.5: VE~ BIE^?U DO^` & TA.O PDF (CHART SERVICE)
- n8n gui code Python (do AI viet) sang "Chart Service".
- Chart Service chay code trong moi truong an toan, ve hinh va gui lai file anh (Base64).
- n8n gom tat ca insights va anh do thanh 1 yeu cau tao PDF.
- Chart Service render PDF xon va tu dong upload len MinIO, tra lai link tai file.

BUO'C 5: TRẢ KẾT QUẢ VÀ BÁO REAL-TIME (CALLBACK & SIGNALR)
- Sau khi n8n hoàn tất (xem chi tiết ở mục 3), nó sẽ gọi về Webhook Callback của API C#.
- API C# thực hiện 2 việc quan trọng:
     1. Cập nhật Database: Chuyển Job sang "Succeeded" và lưu link PDF.
     2. Đẩy SignalR: Gửi tin nhắn thực tế tới đúng nhóm "jobId" đang chờ.
- [LƯU Ý]: n8n KHÔNG trực tiếp đẩy SignalR. n8n chỉ là "người làm thuê", C# mới là "người báo tin".

--------------------------------------------------------------------------------
3. CHI TIẾT WORKFLOW N8N (8 NODE CHỦ CHỐT)
--------------------------------------------------------------------------------
Khi n8n nhận lệnh (Webhook), nó thực hiện chuỗi hành động sau:
1. Webhook (Receive): Tiếp nhận jobId, fileUrl, và prompt từ C#.
2. Download File: Tải file thô từ MinIO về bộ nhớ n8n.
3. Extract Data: Dùng node "Spreadsheet File" để parse Excel/CSV thành JSON.
4. Format Data (Code): Lọc lấy 100 dòng đầu để "mớm" cho AI (Tiết kiệm Token).
5. Gemini 1 (AI): Phân tích schema và tự viết Python Code để vẽ biểu đồ.
6. Execute Chart: Gửi Python Code sang "Chart Service" để render ảnh Base64.
7. Gemini 2 (AI): Dựa trên số liệu thống kê để viết Insights (nhận xét) tiếng Việt.
8. Build PDF & Callback: Gom ảnh + Insights tạo file PDF và ký tên HMAC gửi về C#.

--------------------------------------------------------------------------------
4. VÌ SAO PHẢI LÀM PHỨC TẠP NHƯ THẾ?
--------------------------------------------------------------------------------
- Tránh treo Server: Phân tích file lớn có thể mất 10 phút, việc chay ngầm giúp Server luôn san sàng nhận lệnh khác.
- Trải nghiệm cao cấp: Người dùng không phải F5 trang web, kết quả tự động hiện ra nhờ SignalR.
- Bảo mật tuyệt đối: n8n và API giao tiếp qua chữ ký HMAC-SHA256, không ai có thể giả danh kết quả gửi về cho Server của bạn.
